# Editorial Calendar Template

> Calendar structure and planning framework. Load when scheduling content.

## Calendar Structure

### Weekly View

```markdown
## Week of {date}

### Monday
- [ ] Blog: {title} -- Status: {planned/draft/ready/published}
- [ ] Social: LinkedIn text post -- Topic: {topic}
- [ ] Social: X thread -- Topic: {topic}

### Tuesday
- [ ] Blog: {title} -- Status: {status}
- [ ] Social: LinkedIn carousel -- Topic: {topic}
- [ ] Social: IG carousel -- Topic: {topic}
- [ ] Social: X tweets x3

### Wednesday
- [ ] Newsletter: {subject line} -- Status: {status}
- [ ] Social: LinkedIn text post
- [ ] Social: X tweets x3

### Thursday
- [ ] Blog: {title} -- Status: {status}
- [ ] Social: LinkedIn carousel
- [ ] Social: IG post
- [ ] Social: X thread

### Friday
- [ ] Social: LinkedIn text post (engagement/question)
- [ ] Social: IG reel
- [ ] Social: X tweets x3
```

## Content Mix Targets

| Type | Target Per Week | Pillar Allocation |
|------|----------------|-------------------|
| Blog posts | 3-4 | 2 evergreen SEO, 1 thought leadership, 1 timely |
| LinkedIn posts | 7-10 | Mix of text, carousel, engagement |
| X posts | 15-25 | Mix of threads, tweets, engagement |
| IG posts | 5-7 | Mix of carousels, reels, stories |
| Newsletter | 1 | Curated insights + one original take |

## Pillar Balance

Target distribution across content pillars:

| Category | Percentage | Purpose |
|----------|-----------|---------|
| Evergreen SEO | 60% | Compound organic traffic over time |
| Timely / Trend | 20% | Social engagement, relevance |
| Thought Leadership | 20% | Authority building, differentiation |

Review balance monthly. Adjust if one pillar is overrepresented.

## Planning Cadence

| When | Activity | Who |
|------|---------|-----|
| Monthly | Set monthly theme and pillar priorities | editor-in-chief |
| Weekly (Monday) | Review last week performance, plan this week | editor-in-chief + team |
| Daily | Check calendar, confirm scheduled content, engage | social-media |
| Friday | Quick performance check, adjust next week | editor-in-chief |

## Calendar Management Rules

1. **4-week lookahead minimum** -- never scramble for content
2. **Buffer content** -- maintain 2-3 ready-to-publish pieces for emergencies
3. **Seasonal planning** -- map major events and holidays 8 weeks ahead
4. **Gap detection** -- run `/schedule --gaps` weekly to catch empty slots
5. **Cancellation protocol** -- if a piece is killed, replace within 48 hours

## Content Status Definitions

| Status | Meaning | Next Step |
|--------|---------|-----------|
| Planned | Briefed, not started | Writer begins research |
| In Progress | Writer working on draft | Submit draft by deadline |
| Draft Review | Draft submitted to editor | Editor reviews within 24h |
| Revisions | Editor returned with feedback | Writer revises within 24h |
| Ready | Approved, awaiting publish date | Designer creates visuals |
| Scheduled | Content + visuals ready, scheduled | Auto-publish at set time |
| Published | Live | Social media promotes, monitor 48h |
| Analyzed | 7-day performance reviewed | Insights captured for next cycle |
